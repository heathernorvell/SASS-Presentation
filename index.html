Why use SCSS?

SCSS uses the normal CSS Syntax, but with added functionality and the ability to import multiple files into one.

SCSS Folder Structure Standard

A General rule to thumb is to layout your stylesheets like this: 

stylesheets/
|
|-- admin/           # Admin sub-project
|   |-- modules/
|   |-- partials/
|   `-- _base.scss
|
|-- account/         # Account sub-project
|   |-- modules/
|   |-- partials/
|   `-- _base.scss
|
|-- site/            # Site sub-project
|   |-- modules/
|   |-- partials/
|   `-- _base.scss
|
|-- vendor/          # CSS or Sass from other projects
|   |-- _colorpicker-1.1.scss
|   |-- _jquery.ui.core-1.9.1.scss
|   ...
|
|-- admin.scss       # Primary stylesheets for each project
|-- account.scss
`-- site.scss

Mixins

A mixin can be used anywhere within the system and basically acts as a variable.

Mixins are best for colors or numbers that you know you're going to be using multiple times within the applicaiton.

$mixin-black: #000000;

Directives

With SCSS you can create for, if, foreach, and while loops to vary your CSS. For example:

$class-slug: for !default;

@for $i from 1 through 4{
  .#{$class-slug}-#{$i}{
    width: 60px + $i;
	}
}

Becomes:

.for-1 {
  width: 61px;
}
.for-2 {
  width: 62px;
}
.for-3 {
  width: 63px;
}
.for-4 {
  width: 64px;
}

Functions

With SASS you can define functions to be used throughout the CSS

@mixin my-padding-mixin($some-number) {
  padding: $some-number;
}

Then to use your function:

.my-module {
  @include my-padding-mixin(10px);
}

Compiled:

.my-module {
  padding: 10px;
}
